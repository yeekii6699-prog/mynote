## 门店数字化运营方案摘要
- **核心理念**：去 SaaS 化，使用飞书多维表作为云端数据库，以手机号为身份锚点，前台执行 SOP + 后台 Python RPA（“绿野仙踪模式”）。
- **技术栈**：
  - 飞书多维表：客户档案、预约流水、人员排班、Dashboard、录入表单。
  - Python 本地脚本：feishu-sdk 读写 + pyautogui / uiautomation 控制 PC 微信。
  - PC 微信：执行加好友、改备注、发欢迎语 / 定位。
- **关键数据表**：
  1. 客户档案表（手机号主键，关联备注、绑定状态、渠道、沉睡状态）。
  2. 预约流水表（记录业务、紧急程度、状态、关联客户，触发脚本动作）。
  3. 人员排班表（技师班次 / 在岗状态，防撞单并用于业绩统计）。
- **业务流程**：
  - 场景 A（电话 / 虚拟号）前台录入 → Python 轮询 → 自动加好友 + 改备注 + 回写状态 → 发送定位。
  - 场景 B（扫码 / 被动）人工通过好友 → 询问手机号 → 飞书自动建档 / 关联，保持身份一致。
- **功能亮点**：统一身份、资产私有化、防撞单、沉睡客户唤醒、自动回访。
- **Roadmap**：P1 建表 → P2 飞书 API → P3 微信 RPA → P4 串联闭环 → P5 打包部署与前台培训。
