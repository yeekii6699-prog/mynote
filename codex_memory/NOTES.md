- **技术栈**：
  - 飞书多维表：客户档案 / 预约流水 / 人员排班 + Dashboard + 表单。
  - Python 脚本：feishu-sdk 读写、pyautogui/uiautomation 控制 PC 微信。
  - PC 微信：执行加好友、改备注、发定位等动作。
- **数据库设计**：
  - 客户档案表：手机号主键，记录备注、绑定状态、渠道、沉睡状态等。

  - 人员排班表：管理技师班次，避免撞单并计算业绩。
- **业务流程**：
  - 场景 A（电话/虚拟号）：前台录入 → Python 轮询 → 自动加好友/改备注/回写状态 → 发送定位。
  - 场景 B（扫码/被动）：人工通过好友 → 获取手机号 → 飞书自动建档或关联。
- **功能亮点**：身份统一、资产私有化、防撞单、沉睡客户自动唤醒。
