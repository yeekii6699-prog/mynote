# 门店数字化方案要点

## 核心设计理念
- **去 SaaS 化**：不采购昂贵美业系统，改用飞书多维表作为云端数据库。
- **手机号锚点**：通过强制录入手机号，解决虚拟号/昵称不一致问题，实现身份统一。
- **“绿野仙踪”模式**：前台人工填表（SOP） + 后台 Python 脚本自动执行复杂操作（RPA）。

## 技术架构
- **后端 / 数据层**：飞书多维表，承担客户档案、预约流水、排班数据存储，同时输出 Dashboard 与录入表单。
- **逻辑层**：本地 Python 脚本（feishu-sdk + pyautogui/uiautomation），运行在前台 Windows 电脑，读写飞书数据并驱动 RPA。
- **执行层**：PC 版微信，负责加好友、改备注、发送定位/欢迎语。

## 数据库（关键三张表）
1. 客户档案表：手机号为主键，字段含备注名、绑定状态、来源渠道、沉睡状态等。
2. 预约流水表：记录每笔业务、紧急程度、状态、关联客户，驱动脚本处理。
3. 人员排班表：维护技师班次和在岗状态，自动关联预约防撞单。

## 业务流程
- **场景 A（电话/虚拟号进线）**：前台接电→询问手机号→填飞书表单→勾选“急需定位”。脚本轮询→自动搜号加好友→改备注→回写状态→发送定位。
- **场景 B（扫码/被动进线）**：人工通过好友→获取手机号→飞书自动建档或关联，保持身份统一。

## 功能亮点
- 统一身份与备注，资产私有化存飞书。
- 预约防撞单、排班联动，沉睡客户自动唤醒。
- 支持自动化回访、欢迎语、提醒等高频触点。

## 开发路线（Roadmap）
P1 建表 → P2 飞书 API → P3 微信 RPA → P4 打通闭环 → P5 打包部署 + 前台培训。
